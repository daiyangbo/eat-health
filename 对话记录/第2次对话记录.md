### 阶段二：数据准备（2026-06-14 下午）

#### 2.1 数据收集策略
- 分析了 Eat This Much 的数据来源
- 确定了数据源优先级：
  - 第一优先级：USDA SR Legacy（英文）
  - 第一优先级：USDA Foundation Foods（英文）
  - 第二优先级：中国食物成分表（中文）
  - 第三优先级：Open Food Facts（多语言）

#### 2.2 下载 USDA 数据
- 下载了 USDA SR Legacy 数据集（~30MB）
- 下载了 USDA Foundation Foods 数据集（~3.7MB）
- 创建了数据文件夹和 README 文件

#### 2.3 数据解析
- 编写了 `parse_usda.py` 脚本
- 解析了 7,793 条食物数据
- 提取了蛋白质、脂肪、碳水、热量等核心营养数据

#### 2.4 数据清洗
- 编写了 `clean_data.py` 脚本
- 去除了 1,337 条无效数据
- 最终保留 6,456 条食物数据
- 添加了营养标签（高蛋白、低脂等）

#### 2.5 生成训练数据
- 编写了 `generate_train.py` 脚本
- 生成了 600 条训练数据
- 划分：480 条训练 + 120 条验证

#### 2.6 搭建 RAG 知识库
- 使用 ChromaDB 搭建向量数据库
- 使用 all-MiniLM-L6-v2 作为 Embedding 模型
- 向量化了 6,456 条食物数据
- 测试了检索效果

---

### 阶段三：模型训练（2026-06-14 晚上）

#### 3.1 AutoDL 环境准备
- 注册 AutoDL 账号
- 创建 RTX 4090 GPU 实例
- 扩展存储到 50GB

#### 3.2 克隆 GitHub 仓库
- 使用 `ghfast.top` 镜像加速克隆
- 仓库地址：https://github.com/daiyangbo/eat-health

#### 3.3 安装依赖
- 安装了 torch、transformers 等依赖到 `/root/pip_packages`
- 设置了 PYTHONPATH 环境变量
- 设置了 HuggingFace 镜像

#### 3.4 安装 LLaMA-Factory
- 遇到问题：最新版需要 Python 3.11+
- 解决：使用 v0.9.3 版本（兼容 Python 3.10）
- 使用 `ghfast.top` 镜像加速克隆

#### 3.5 数据格式转换
- 运行 `convert_to_llama_factory.py`
- 生成了 `train_llama.json` 和 `val_llama.json`
- 创建了 `dataset_info.json`

#### 3.6 训练配置
- 遇到问题：全参数微调 OOM
- 解决：使用 LoRA 微调
- 配置：
  - `finetuning_type: lora`
  - `per_device_train_batch_size: 1`
  - `gradient_accumulation_steps: 16`

#### 3.7 执行训练
- 训练时间：4 分 33 秒
- 训练 Loss：1.688（从 1.74 下降到 1.57）
- 验证 Loss：1.566
- 可训练参数：9,232,384（0.6%）
