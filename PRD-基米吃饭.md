> **文档版本**：v1.0 MVP
> **创建日期**：2026-06-10

> **目标用户**：西南大学在校学生（约5万人）

---

## 一、产品概述

### 1.1 产品名称
**基米吃饭**

### 1.2 产品定位
基于大语言模型（LLM）+ RAG 技术的校园智能营养配餐助手，帮助学生根据营养目标快速获取食堂食材推荐。

### 1.3 核心价值
> **一句话描述**：输入三大营养素目标，智能推荐食堂食材搭配

### 1.4 产品愿景
- **短期**：MVP版本，验证 LLM + RAG 在营养配餐场景的可行性
- **中期**：完善功能，支持交互式引导、偏好设置、食材替代
- **长期**：上线微信小程序，服务西南大学全体学生

## 二、用户分析

### 2.1 目标用户画像

| 维度 | 描述 |
|------|------|
| **身份** | 西南大学在校学生 |
| **年龄** | 18-25岁 |
| **规模** | 约5万人 |
| **场景** | 在学校食堂就餐 |
| **痛点** | 选择困难、健身饮食困惑、营养搭配不知 |

### 2.2 用户场景

**场景一：健身学生**
```
小明在健身房练完，想补充蛋白质，但不知道食堂吃什么
→ 打开基米吃饭，输入蛋白质150g、脂肪60g、碳水200g
→ 系统推荐：鸡胸肉200g + 鸡蛋3个 + 米饭300g + 西兰花150g
→ 小明去食堂按推荐打菜
```

**场景二：减脂学生**
```
小红想减脂，控制热量摄入，但不知道怎么搭配
→ 打开基米吃饭，输入减脂目标
→ 系统推荐：低脂高蛋白的食材搭配
→ 小红按推荐就餐
```

**场景三：选择困难学生**
```
小刚每天中午不知道吃什么
→ 打开基米吃饭，输入营养目标
→ 系统推荐具体食材和用量
→ 小刚不用纠结，直接去吃
```

### 2.3 用户痛点分析

| 痛点 | 描述 | 严重程度 | MVP是否解决 |
|------|------|----------|-------------|
| **选择困难** | 食堂菜品多，不知道选什么 | ⭐⭐⭐⭐ | ✅ 是 |
| **健身饮食困惑** | 想增肌/减脂，不知道吃什么 | ⭐⭐⭐⭐⭐ | ✅ 是 |
| **营养搭配不知** | 想控制营养，不知道怎么搭配 | ⭐⭐⭐⭐⭐ | ✅ 是 |

### 4.3 数据方案

#### 4.3.1 训练数据格式

```json
{
  "input": "蛋白质150g，脂肪60g，碳水200g",
  "output": "推荐食材：\n1. 鸡胸肉 200g（蛋白质40g，脂肪4g）\n2. 鸡蛋 3个（蛋白质18g，脂肪15g）\n3. 米饭 300g（碳水78g）\n4. 西兰花 150g（蛋白质4g，碳水10g）\n5. 牛肉 150g（蛋白质30g，脂肪15g）\n6. 糙米 200g（碳水150g）"
}
```

#### 4.3.2 RAG知识库结构

```json
{
  "food_id": 1,
  "name": "鸡胸肉",
  "category": "肉类",
  "serving_size": 100,
  "serving_unit": "g",
  "nutrition": {
    "protein": 20.0,
    "fat": 2.0,
    "carbs": 0.0,
    "calories": 110
  },
  "common_portions": [
    {"amount": 100, "unit": "g"},
    {"amount": 200, "unit": "g"}
  ]
}
```

#### 4.3.3 数据来源

| 数据源 | 内容 | 说明 |
|--------|------|------|
| **中国食物成分表** | 中式食材营养数据 | 标准参考 |
| **USDA食品数据库** | 国际食材营养数据 | 补充数据 |
| **自定义数据** | 食堂特色菜品 | 后续补充 |

### 4.4 模型训练方案

#### 4.4.1 训练流程

```
Step 1: 准备数据
├── 收集公开营养数据集
├── 生成训练数据（输入-输出对）
└── 数据清洗和格式化

Step 2: 搭建RAG知识库
├── 整理食材营养数据
├── 文本向量化（Embedding）
└── 存入向量数据库（ChromaDB）

Step 3: 微调模型
├── 选择基座模型（Qwen2.5-1.5B）
├── 配置训练参数
├── 在AutoDL上进行SFT微调
└── 保存训练后的模型

Step 4: 集成RAG
├── 用户输入营养目标
├── RAG检索相关食材
├── LLM生成推荐结果
└── 格式化输出

Step 5: 测试优化
├── 测试不同输入
├── 评估推荐质量
└── 优化提示词和检索策略
```

#### 4.4.2 训练参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基座模型 | Qwen2.5-1.5B | 1.5B参数 |
| 训练方式 | SFT（全量微调） | 学习完整流程 |
| 学习率 | 1e-5 | 标准值 |
| Batch Size | 4 | 受显存限制 |
| 梯度累积 | 4 | 有效batch=16 |
| Epochs | 3 | 标准值 |
| 最大长度 | 512 | 输入+输出 |
| GPU需求 | RTX 3090 (24GB) | AutoDL提供 |
