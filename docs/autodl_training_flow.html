GPU 对比：

  • RTX 4090：¥2.5/小时，训练更快（推荐）

⏱️ 预计 5 分钟 💰 ¥1.5-2.5/小时

第三阶段：环境配置
4
安装依赖包并设置镜像
在 AutoDL 实例上安装项目所需的 Python 包，并设置 HuggingFace 镜像加速模型下载。
# 进入项目目录
cd /root/eat-health

# 清理缓存（释放 / 目录空间）
pip cache purge

# 安装依赖到 /root 目录（空间更充足）
pip install -r requirements.txt --target /root/pip_packages --no-cache-dir

# 设置 Python 路径
export PYTHONPATH=/root/pip_packages:$PYTHONPATH

# 设置 HuggingFace 镜像（重要！加速模型下载）
export HF_ENDPOINT=https://hf-mirror.com

为什么要安装到 /root 目录？

  • AutoDL 的 / 目录只有 30GB，空间紧张
  • /root 目录有 98GB，空间充足
  • PyTorch + CUDA 依赖约 2-3GB，需要足够空间

为什么要用 --no-cache-dir？

  • 不缓存可以节省 2-3GB 磁盘空间
  • 避免把 / 目录撑满
  • 对后续训练没有影响

为什么要设置 HuggingFace 镜像？

  • Qwen2.5-1.5B 模型约 3GB，直接下载很慢
  • 使用镜像可以加速 5-10 倍
  • 这是国内用户的标准做法
⏱️ 预计 10 分钟 ⚠️ 确保 / 目录有至少 5GB 可用空间
⬇️
5
下载基座模型
下载 Qwen2.5-1.5B 模型到本地，用于后续微调。
# 设置 HuggingFace 镜像（国内加速）
export HF_ENDPOINT=https://hf-mirror.com

# 使用下载脚本（推荐）
python download_model.py

下载脚本功能：

  • 自动检查模型是否已下载
  • 支持重试机制（最多 3 次）
  • 显示下载进度和模型信息
  • 保存到 ./models/base/qwen2.5-1.5b/
⏱️ 预计 10-15 分钟 📦 模型大小约 3GB
⬇️
6
安装 LLaMA-Factory
安装 LLaMA-Factory 微调框架，用于 SFT 训练。需要使用兼容 Python 3.10 的版本。
# 清理旧版本（如果存在）
rm -rf /root/pip_packages/llamafactory
rm -rf /root/eat-health/LLaMA-Factory

# 克隆兼容版本（v0.9.3，支持 Python 3.10）
cd /root/eat-health
git clone --branch v0.9.3 https://ghfast.top/https://github.com/hiyouga/LLaMA-Factory.git

# 安装到系统目录
cd LLaMA-Factory
pip install -e . --no-cache-dir

为什么要用 v0.9.3 版本？

  • 最新版 LLaMA-Factory 需要 Python 3.11+
  • AutoDL 默认是 Python 3.10，不兼容
  • v0.9.3 支持 Python 3.10，稳定可靠

为什么要用镜像站？

  • GitHub 直连很慢，容易超时
  • ghproxy.net 是 GitHub 镜像站，速度快
⏱️ 预计 5-10 分钟
⬇️
📊
第四阶段：数据准备
7
转换训练数据格式
将 train.json 转换为 LLaMA-Factory 支持的格式，并创建 dataset_info.json 配置文件。
# convert_to_llama_factory.py
python convert_to_llama_factory.py

转换前后对比：

  • 输入格式：{"input": "蛋白质150g...", "output": "推荐食材..."}
  • 输出格式：{"instruction": "...", "input": "...", "output": "..."}

生成的文件：

  • train_llama.json - LLaMA-Factory 格式的训练数据
  • val_llama.json - LLaMA-Factory 格式的验证数据
  • dataset_info.json - 数据集配置文件（LLaMA-Factory 必需）
⏱️ 预计 2 分钟
⬇️
🎯
第五阶段：模型训练
8
执行 SFT 微调
使用 LLaMA-Factory 进行监督微调训练。
# 进入项目根目录
cd /root/eat-health

# 运行训练（使用 Python 模块方式）
python -m llamafactory.cli train /root/eat-health/train_config.yaml

为什么要用 python -m 方式？

  • llamafactory-cli 命令可能找不到
  • 使用 Python 模块方式更可靠
  • 需要在项目根目录运行

训练参数详解：

参数 值 说明
model_name_or_path ./models/base/qwen2.5-1.5b 基座模型路径
num_train_epochs 3 训练轮数（SFT 推荐 3 轮）
per_device_train_batch_size 4 每个设备的批次大小
gradient_accumulation_steps 4 梯度累积步数（有效批次 = 4×4 = 16）
learning_rate 1e-5 学习率（SFT 推荐 1e-5 到 2e-5）
lr_scheduler_type cosine 学习率调度器
warmup_steps 100 预热步数
bf16 true 使用 bfloat16 精度（节省显存）
save_steps 500 每 500 步保存一次 checkpoint
logging_steps 10 每 10 步记录一次日志
⏱️ 预计 2-3 小时 💰 约 ¥6-9
⬇️

第六阶段：模型评估
9
测试模型效果
使用测试数据评估模型的推荐效果。
# test_model.py
python test_model.py

测试用例：

  • 输入：蛋白质150g，脂肪60g，碳水200g
  • 预期：输出包含至少5种食材，营养数据合理

评估指标：

  • 推荐合理性：推荐的食材是否符合营养目标
  • 营养准确性：推荐的营养数据是否准确
  • 多样性：推荐的食材种类是否多样
  • 响应时间：生成推荐的时间
⏱️ 预计 30 分钟

🔧 常见问题及解决方案

问题 1：CUDA 内存不足

RuntimeError: CUDA out of memory

解决方案：
1. 减小批次大小：修改 train_config.yaml 中的 per_device_train_batch_size: 2
2. 增加梯度累积：gradient_accumulation_steps: 8
3. 使用梯度检查点：添加 gradient_checkpointing: true

问题 2：GitHub 克隆超时

fatal: unable to access: Connection timed out

解决方案：
1. 开启学术加速：source /etc/network_turbo
2. 使用镜像站克隆：git clone https://ghfast.top/https://github.com/daiyangbo/eat-health.git
3. 如果还是慢，等几分钟再试

问题 3：磁盘空间不足（No space left on device）

OSError: [Errno 28] No space left on device

原因：AutoDL 的 / 目录只有 30GB，PyTorch + CUDA 依赖约 2-3GB，加上缓存可能撑满。

解决方案：
1. 扩展存储：在 AutoDL 控制台扩展到 50GB 以上
2. 清理缓存：pip cache purge
3. 安装到 /root 目录：pip install -r requirements.txt --target /root/pip_packages --no-cache-dir
4. 设置 Python 路径：export PYTHONPATH=/root/pip_packages:$PYTHONPATH

问题 4：模型下载失败

ConnectionError 或 TimeoutError

解决方案：
1. 设置 HuggingFace 镜像：export HF_ENDPOINT=https://hf-mirror.com
2. 确保已开启学术加速：source /etc/network_turbo
3. 使用代理
4. 手动下载模型并上传到 ./models/base/qwen2.5-1.5b/

问题 5：LLaMA-Factory 安装失败

ERROR: Package 'llamafactory' requires a different Python: 3.10.8 not in '>=3.11.0'

原因：最新版 LLaMA-Factory 需要 Python 3.11+，但 AutoDL 默认是 Python 3.10。

解决方案：
1. 使用兼容版本：git clone --branch v0.9.3 https://ghfast.top/https://github.com/hiyouga/LLaMA-Factory.git
2. 清理旧版本：rm -rf /root/pip_packages/llamafactory
3. 安装到系统目录：pip install -e . --no-cache-dir

问题 6：transformers 版本冲突

ImportError: transformers>=4.45.0,<=4.52.4 is required

解决方案：
1. 跳过版本检查：export DISABLE_VERSION_CHECK=1
2. 或降级 transformers：pip install transformers==4.52.4

问题 7：训练速度慢

解决方案：
1. 确认使用了 GPU（检查 nvidia-smi）
2. 增加 dataloader_num_workers
3. 使用混合精度训练（已启用 bf16）

问题 8：训练 Loss 不下降

解决方案：
1. 检查数据格式是否正确
2. 调整学习率（尝试 2e-5 或 5e-6）
3. 增加训练轮数
4. 检查数据质量

问题 9：显存占用过高

解决方案：
1. 使用 nvidia-smi 监控显存
2. 减小 max_length（如果设置了的话）
3. 使用梯度检查点：gradient_checkpointing: true

问题 10：llamafactory-cli 命令找不到

/bin/sh: 1: llamafactory-cli: not found

解决方案：
使用 Python 模块方式运行：python -m llamafactory.cli train /root/eat-health/train_config.yaml
