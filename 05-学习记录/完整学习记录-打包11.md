### Part 2：RLHF（P7-13）

#### 2.1 强化学习RL基础概念（P7-11）

| 概念 | 定义 | 比喻 |
|------|------|------|
| Agent（智能体） | 感知环境状态，根据奖励选择动作，最大化长期收益 | 学生 |
| Environment（环境） | 接收动作，评价并反馈可量化信号 | 老师/考试 |
| Policy（策略） | 状态→动作的映射，定义智能体在时刻t选择行为的方式 | 解题方法 |
| Reward Function（奖励函数） | 每步环境中向智能体发送的收益信号 | 考试分数 |
| Value Function（价值函数） | 从长远角度看什么是好的，状态开始的累积总收益期望 | 长期投资回报 |
| Environment Model（环境模型） | 对环境反应模式的模拟，允许推断外部环境行为 | 预测题型 |

#### 2.2 RLHF核心思想（P12-13）

**问题**：Native RL有Environment和Reward Model，但大语言模型强化学习没有奖励函数，只有人类/专家的示范，怎么办？

**解决方案**：
```
人类标注数据 → 训练Reward Model → 反推人类的奖励函数 → 用RL方法找最优策略
```

关键洞察：通过人类标注数据训练得到Reward Model（相当于有了人类标注数据，相信它是不错的，然后反推人类因为什么样的奖励函数才会采取这些行为），有了奖励函数之后，就可以使用一般的强化学习方法去找出最优策略/动作。

**OpenAI的大语言模型构建流程**：
```
预训练 → SFT → RM训练 → PPO强化学习 → 对齐后的模型
```

### Part 3：PPO（P15-19）

#### 3.1 强化学习方法分类（P15）

| 方法 | 特点 | 代表算法 |
|------|------|---------|
| Value-based | 学习价值函数，隐式策略 | Q-Learning、DQN |
| Policy-based | 无价值函数，直接学习策略 | REINFORCE |
| Actor-Critic | 结合两者 | PPO |

#### 3.2 策略梯度下降PG（P16-17）

- Value-based：比较a1/a2/a3的Q-value，选最大的
- Policy-based：有一个计算动作概率的函数(actor)，根据概率p(s,a)选取动作
- PG优势：不输出Q值，直接输出动作概率，能处理连续动作空间

PG训练流程：
```
Actor与Environment互动 → 产生采样数据(Trajectory) → 获得(s,a)对 → 计算R(τ) → 更新参数θ
更新公式：θ ← θ + η∇R̄_θ
```

PG的问题：策略参数更新后，需要重新与环境互动收集数据再进行下一轮迭代（采样效率低）

#### 3.3 PPO算法原理（P18-19）

**核心创新1——重要性采样**：
- 在不知道策略路径概率p的情况下，通过模拟近似分布q
- 只要p和q分布差距不大，多轮迭代可快速收敛

**核心创新2——近端约束（Clipping）**：
```
PPO目标函数:
  L = min(r(θ) × A, clip(r(θ), 1-ε, 1+ε) × A)
  r(θ) = π_new(a|s) / π_old(a|s)  ← 概率比
  A = advantage（优势函数）         ← 这个动作比平均好多少
  ε = 0.2                          ← 裁剪范围
```

**Actor-Critic架构**：
| 组件 | 职责 | 对应 |
|------|------|------|
| Actor | 与环境互动收集样本，PPO梯度更新 | 策略网络（GPT-2） |
| Critic | 评判Actor的动作好坏 | 价值网络（Value Head） |

## 二、核心知识点详解

### Q1: RLHF是什么？

**RLHF = Reinforcement Learning from Human Feedback（基于人类反馈的强化学习）**

核心问题：大模型预训练目标是"预测下一个token"，不是"说有用的话"。预训练后的模型会续写文本，但不会按人类意图回答问题。

三阶段流程：
```
阶段1: 预训练（Pre-training）
  目标: 学会语言能力
  数据: 互联网海量文本
  结果: 会说话，但不会对话

阶段2: 监督微调（SFT）
  目标: 学会按指令回答
  数据: 人工编写的"指令-回答"对
  结果: 会对话，但质量参差不齐

阶段3: RLHF（本章重点）
  目标: 让输出符合人类偏好
  数据: 人类对多个回答的排序/评分
  结果: 输出高质量、安全、有用的回答
```

### Q2: RLHF的三个子步骤

```
Step 1: 收集人类偏好数据
  问题: "地球是平的吗？"
  回答A: "是的，地球是平的。"        ← 人类排序: C > B > A
  回答B: "地球不是平的，是球形。"
  回答C: "地球是近球形，赤道略鼓。"

Step 2: 训练奖励模型（Reward Model）
  输入: (问题, 回答)
  输出: 标量分数
  训练目标: 分数排序 = 人类排序
  C(0.9) > B(0.6) > A(0.2) ✓

Step 3: PPO强化学习
  1. 模型生成回答
  2. RM给回答打分（reward）
  3. PPO根据reward更新模型
  4. 重复，直到模型学会生成高分回答
```

### Q3: 为什么需要RLHF而不是直接SFT？

| | SFT | RLHF |
|--|-----|------|
| 训练信号 | "标准答案"（模仿） | "哪个更好"（偏好） |
| 数据需求 | 要写出完美答案（难） | 只需要排序（容易） |
| 能学到什么 | 学会"怎么写" | 学会"写什么才好" |
| 局限 | 标注员写不出最优答案 | 标注员能判断哪个更好 |

关键洞察：让人类写出"完美回答"很难，但让人类判断"哪个回答更好"很容易。RLHF利用了这个不对称性。

### Q4: PPO的核心创新

**创新1：重要性采样（Importance Sampling）**
```
PG的问题: 旧策略收集的数据，新策略不能用
PPO的解法: 用重要性采样"修正"旧数据

修正公式:
  E_π_new [reward] ≈ E_π_old [reward × π_new(a|s) / π_old(a|s)]
```

**创新2：近端约束（Clipping）**
```
如果 r(θ) 超出 [0.8, 1.2] → 被裁剪，不允许更新太大
如果 r(θ) 在 [0.8, 1.2] 内 → 正常更新
```

直觉：每次只允许模型"小步走"，不能一步跨太大，避免走偏。

### Q5: 为什么需要KL散度约束？

没有KL约束会发生什么——Reward Hacking：
```
奖励模型喜欢"积极"词汇 → GPT-2学会疯狂堆砌"good good good"
RM分数很高 → 但生成的文本毫无意义

KL约束的作用:
  "你可以变，但不能变得太离谱"
  偏离参考模型太远 → 惩罚 → 迫使模型在"拿高分"和"保持正常"之间平衡
```

### Q6: PPO在RLHF中的完整流程

```
一次PPO迭代：
  ① Rollout（生成）
     query: "The movie was"
     GPT-2 → response: "absolutely fantastic and moving"

  ② Evaluation（评估）
     query + response → Reward Model → score = 2.8

  ③ 计算log概率
     model(query+response) → logprob_new
     ref_model(query+response) → logprob_ref

  ④ 计算KL散度
     KL = logprob_new - logprob_ref
     KL大 → model偏离太远 → 惩罚

  ⑤ 计算最终reward
     final_reward = score - β × KL

  ⑥ PPO更新
     用final_reward计算梯度，更新model参数
     clip机制确保更新幅度不超过ε
```

### Q7: 本实验的角色映射

| RL概念 | 本实验中的对应 | 代码 |
|--------|-------------|------|
| Agent/Actor | GPT-2（策略模型） | `model` |
| Environment | 用户query | `batch["query"]` |
| Action | 生成的response | `batch["response"]` |
| Reward | BERT情感分数 | `rewards` |

| Reference Policy | 参考模型 | `ref_model` |
| KL约束 | TRL自动计算 | `ppo_trainer.step()` 内部 |

### Q8: 实验的核心目的

用PPO让GPT-2学会一个特定偏好：生成积极评论。

注意"提升质量"不够准确。PPO提升的不是"质量"，而是对齐人类偏好。
```
训练前的GPT-2：续写评论 → 可能积极，可能消极，随缘
训练后的GPT-2：续写评论 → 几乎都是积极的
```

模型的语言能力没变（没学新知识），变的是输出倾向——从"随便写"变成了"偏向写积极的"。

这就是ChatGPT和普通GPT-4的核心区别：
| | 普通预训练模型 | RLHF后的模型 |
|--|--------------|-------------|
| 问"地球是平的吗" | 可能顺着你说"是的" | 会纠正你"不是，地球是近球形" |
| 问"帮我写代码" | 可能写有毒代码 | 会拒绝或加安全提示 |

本质：预训练让模型"会说话"，SFT让模型"听懂指令"，RLHF让模型"说人话、有分寸"。

### 核心代码片段

#### 数据加载（Cell 14-15）

```python
def build_dataset(config, dataset_name="dataset/imdb",
                  input_min_text_length=2, input_max_text_length=8):
    tokenizer = AutoTokenizer.from_pretrained(config.model_name)
    tokenizer.pad_token = tokenizer.eos_token
    ds = load_dataset(dataset_name, split="train")
    ds = ds.rename_columns({"text": "review"})
    ds = ds.filter(lambda x: len(x["review"]) > 200, batched=False)
    input_size = LengthSampler(input_min_text_length, input_max_text_length)
    def tokenize(sample):
        sample["input_ids"] = tokenizer.encode(sample["review"])[: input_size()]
        sample["query"] = tokenizer.decode(sample["input_ids"])
        return sample
    ds = ds.map(tokenize, batched=False)
    ds.set_format(type="torch")
    return ds

dataset = build_dataset(config)
def collator(data):
    return dict((key, [d[key] for d in data]) for key in data[0])
```

数据处理流程：
```
原始IMDB评论（50K条）
    ↓
筛选：只保留 > 200字符的评论
    ↓
分词：tokenizer.encode(review)
    ↓
随机截断：取前2~8个token作为query（评论开头）
    ↓
解码：tokenizer.decode → 得到query文本
    ↓
输出：{"input_ids": tensor, "query": "Well I guess I know"}
```

#### 模型加载（Cell 17）

```python
model = AutoModelForCausalLMWithValueHead.from_pretrained(config.model_name)
ref_model = AutoModelForCausalLMWithValueHead.from_pretrained(config.model_name)
tokenizer = AutoTokenizer.from_pretrained(config.model_name)
tokenizer.pad_token = tokenizer.eos_token
```

AutoModelForCausalLMWithValueHead的结构：
```
普通GPT-2的输出：hidden_states → logits（预测下一个token的概率）
加了Value Head的输出：hidden_states → logits + value（状态价值估计）

Value Head就是一个线性层：hidden_size → 1
它输出一个标量，表示"当前状态有多好"（Critic的角色）
```

#### PPO训练循环（Cell 30）

```python
output_min_length = 4
output_max_length = 16
output_length_sampler = LengthSampler(output_min_length, output_max_length)

generation_kwargs = {
    "min_length": -1, "top_k": 0.0, "top_p": 1.0,
    "do_sample": True, "pad_token_id": tokenizer.eos_token_id,
}

for epoch, batch in enumerate(tqdm(ppo_trainer.dataloader)):
    query_tensors = batch["input_ids"]

    # Step 1: GPT-2生成response
    response_tensors = []
    for query in query_tensors:
        gen_len = output_length_sampler()
        generation_kwargs["max_new_tokens"] = gen_len
        query_response = ppo_trainer.generate(query, **generation_kwargs).squeeze()
        response_len = len(query_response) - len(query)
        response_tensors.append(query_response[-response_len:])
    batch["response"] = [tokenizer.decode(r.squeeze()) for r in response_tensors]

    # Step 2: BERT计算reward
    texts = [q + r for q, r in zip(batch["query"], batch["response"])]
    pipe_outputs = sentiment_pipe(texts, **sent_kwargs)
    positive_scores = [
        item["score"] for output in pipe_outputs
        for item in output if item["label"] == "POSITIVE"
    ]
    rewards = [torch.tensor(score) for score in positive_scores]

    # Step 3: PPO更新
    stats = ppo_trainer.step(query_tensors, response_tensors, rewards)
    ppo_trainer.log_stats(stats, batch, rewards)
```

每一步做三件事：
```
Step 1: Rollout（生成）
  query: "Well I guess I know"
  GPT-2 generate (4~16 tokens) → response: "that Cantor may be an"

Step 2: Evaluation（打分）
  text = query + response → distilbert-imdb → reward = POSITIVE score

Step 3: Optimization（PPO更新）
  ppo_trainer.step() 内部：
  1. model前向传播 → 新logprob
  2. ref_model前向传播 → 旧logprob
  3. 计算KL散度 = 新logprob - 旧logprob
  4. advantage = reward - KL惩罚
  5. PPO梯度更新model参数
  6. 更新Value Head
```

## 八、术语表

| 术语 | 全称 | 含义 |
|------|------|------|
| RLHF | Reinforcement Learning from Human Feedback | 基于人类反馈的强化学习 |
| PPO | Proximal Policy Optimization | 近端策略优化算法 |
| PG | Policy Gradient | 策略梯度下降 |
| RM | Reward Model | 奖励模型 |
| Actor | - | 策略网络，负责生成动作（GPT-2） |
| Critic | - | 价值网络，负责评估状态价值（Value Head） |
| KL散度 | Kullback-Leibler Divergence | 两个概率分布的差异度量 |
| Advantage | - | 优势函数，动作比平均水平好多少 |
| Clip | - | PPO的裁剪机制，限制更新幅度 |
| Importance Sampling | - | 重要性采样，用旧数据近似新策略 |
| Reward Hacking | - | 模型钻奖励函数漏洞 |
| Value Head | - | 给模型加的线性层，输出状态价值估计 |
| Trajectory | - | 轨迹，Agent与环境交互的完整序列 |
| Rollout | - | 生成阶段，模型根据query生成response |
| TRL | Transformer Reinforcement Learning | HuggingFace的RL训练框架 |
