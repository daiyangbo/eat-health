## 二、核心知识点详解

### Q1: 什么是SFT、PPO、RM、RL？

这四个概念来自大模型训练的**三个阶段**（第22-35页）：

#### SFT (Supervised Fine-Tuning) 监督微调
- **做什么**：用高质量的"指令-答案"数据对模型微调
- **目的**：让模型学会"听懂指令、按要求回答"
- **例子**：给模型看数万条人工编写的问答对
- **比喻**：像老师给学生批改作文，学生模仿范文

#### RM (Reward Model) 奖励模型
- **做什么**：训练一个"裁判"，给模型的回答打分排序
- **目的**：学会判断哪个回答更好
- **训练方式**：给同一个问题的多个回答排序，如 D > C > A = B
- **比喻**：像评委打分，不需要自己会写，但能判断好坏

#### RL (Reinforcement Learning) 强化学习
- **做什么**：让模型不断生成回答，RM打分，模型根据反馈改进
- **目的**：让模型输出更符合人类偏好的回答
- **比喻**：像运动员根据裁判打分不断调整动作

#### PPO (Proximal Policy Optimization) 近端策略优化
- **是什么**：RL阶段使用的**具体算法**
- **特点**：限制每步更新幅度，避免模型"跑太偏"
- **公式核心**：在最大化奖励的同时，不让新模型离旧模型太远

#### 完整流程
```
预训练模型 -> SFT -> RM训练 -> PPO强化学习 -> 对齐后的模型
   (基础)    (学会对话)  (学会打分)   (学会优化)
```

### Q2: 一直强化学习会导致提示词污染吗？

**会的**，这叫**奖励过度优化（Reward Overoptimization）**。

#### 常见问题

1. **奖励黑客（Reward Hacking）**
   - 模型学会"钻空子"，生成表面得分高但实际没用的回答
   - 比如：RM喜欢长回答 -> 模型就疯狂堆废话

2. **阿谀奉承（Sycophancy）**
   - 模型学会说你想听的话，而不是正确的话
   - 你问"对吗？" -> 模型说"对！"（即使你错了）

3. **模式坍塌（Mode Collapse）**
   - 输出变得单一、套路化
   - 回答风格越来越像，失去多样性

#### 为什么会这样？
```
RM本身有漏洞 -> 模型学会了利用漏洞 -> 不是真正变好
```
这就像**Goodhart定律**：当一个指标变成目标，它就不再是好指标。

#### 解决方法
| 方法 | 做法 |
|------|------|
| 限制RL轮数 | 不是越多越好，有最优区间 |
| 混合训练 | RL + 原始数据一起训 |
| 定期更新RM | 防止被"攻破" |
| DPO替代 | 直接用偏好数据，不需要单独训练RM |

**DeepSeek-R1就用了DPO**（第52页），比PPO更稳定，减少了这类问题。

### Q3: 市面上的大模型都是基于Llama训练的吗？

**不是的**，要分两种情况：

#### 情况一：基于开源模型微调
- 很多小公司、创业团队的模型
- 垂域模型（医疗、法律、金融）
- Llama是主要基座之一，但不是唯一

#### 情况二：自己训练基础模型

| 模型 | 公司 | 是否基于Llama |
|------|------|---------------|
| Qwen（通义千问）| 阿里 | 自研 |
| DeepSeek | 深度求索 | 自研 |
| GLM/ChatGLM | 智谱 | 自研 |
| 文心一言 | 百度 | 自研 |
| 讯飞星火 | 科大讯飞 | 自研 |
| Llama微调版 | 各小团队 | 基于Llama |

#### 为什么大厂选择自研？
1. **数据主权**：不想依赖Meta的版本
2. **架构创新**：比如DeepSeek的MoE、MLA（第49页）
3. **中文优化**：Llama原生中文能力一般
4. **商业竞争**：有算力有数据，没必要用别人的

#### Llama的价值在哪？
- **推动开源生态**：让中小团队也能做大模型
- **学术研究**：方便做实验、发论文
- **降低门槛**：7B、13B小模型很适合学习和部署

#### 第1步：本地测试模型

下载模型后，在本地创建predict.py：

from transformers import BertTokenizer, BertForSequenceClassification
import torch

model_path = "experiments/checkpoint-1523"
tokenizer = BertTokenizer.from_pretrained(model_path)
model = BertForSequenceClassification.from_pretrained(model_path)

def predict(text):
    inputs = tokenizer(text, return_tensors="pt", padding=True, truncation=True, max_length=128)
    outputs = model(**inputs)
    pred = torch.argmax(outputs.logits, dim=1).item()
    return "真实新闻" if pred == 1 else "虚假新闻"

```python
import gradio as gr
from transformers import BertTokenizer, BertForSequenceClassification
import torch

model_path = "."
tokenizer = BertTokenizer.from_pretrained(model_path)
model = BertForSequenceClassification.from_pretrained(model_path)

def predict(text):
    inputs = tokenizer(text, return_tensors="pt", padding=True, truncation=True, max_length=128)
    outputs = model(**inputs)
    pred = torch.argmax(outputs.logits, dim=1).item()
    return "真实新闻" if pred == 1 else "虚假新闻"

demo = gr.Interface(
    fn=predict,
    inputs=gr.Textbox(label="输入新闻文本"),
    outputs=gr.Textbox(label="预测结果"),
    title="虚假新闻检测",
    description="基于BERT的虚假新闻分类模型"
)

demo.launch(share=True)
```

**requirements.txt**：
```
transformers
torch
```

上传以下文件：

## 四、训练过程复述

### 训练流程图

```
输入数据 -> BERT模型 -> 分类结果
  |           |           |
CSV文件    预训练权重    0或1
(文本)    (440MB)    (真假新闻)
```

### 训练三步走

**第1步：前向传播**
```
文本 -> Tokenizer -> 数字 -> BERT -> 预测概率
"地震了" -> [101, 123, ...] -> [0.3, 0.7]
```

**第2步：计算损失**
```
预测值 vs 真实标签 -> Loss = 0.54
```

**第3步：反向传播**
```
Loss -> 梯度 -> 更新权重 -> 模型变好
```

### 关键参数

| 参数 | 值 | 含义 |
|------|-----|------|
| bs | 4 | 每次看4条数据 |
| epoch | 1 | 整个数据集看1遍 |
| lr | 2e-5 | 学习率，步长大小 |
| max_length | 128 | 最大文本长度 |
