### 1. 知识编辑简介
- 什么是知识编辑：在不重新训练模型的情况下，精准修改模型记住的某一条知识
- 为什么需要：大模型训练成本高，不能因为一条错误知识就重新训练
- 核心工具：EasyEdit（浙大zjunlp团队开源）

### 2. EasyEdit框架
- **Editor**：编辑场景入口，包含待编辑的模型和知识
- **Method**：具体编辑算法（ROME、MEMIT、MEND等）
- **Evaluate**：评估指标（可靠性、通用性、局部性、可移植性）
- **Trainer**：部分方法需要的训练模块

### 3. 编辑案例
- 目标：将GPT-2-XL中梅西的职业从足球改成篮球
- 方法：ROME（Rank-One Model Editing）
- 数据四要素：prompt、ground_truth、target_new、subject

### 4. 评估指标
- Reliability（可靠性）：编辑目标是否成功
- Generality（通用性）：换种问法能否答对
- Locality（局部性）：不相关知识是否被破坏
- Portability（可移植性）：相关推理是否也更新

### 5. 规模化编辑
- Batch edit：同时编辑多条知识，推荐MEMIT方法
- Benchmark测试：Counterfact、ZsRE标准数据集

## 二、核心知识点详解

### Q1: 什么是ROME方法？

**ROME = Rank-One Model Editing（秩一模型编辑）**

#### 核心思想
模型的知识存储在权重矩阵中。ROME认为某条知识主要存储在模型的MLP层中，只需要修改一个秩一矩阵（rank-one update），就能改变这条知识。

#### 数学表达
```
W_new = W_old + ΔW
其中 ΔW = u ⊗ v = u · v^T（秩一矩阵）
```

- **u（左向量）**：表示"改哪里"——定位到存储该知识的神经元
- **v（右向量）**：表示"改成什么"——新的知识内容

#### 与梯度下降的区别

| | 梯度下降（微调） | ROME（秩一更新） |
|--|--|--|
| 方式 | 迭代优化，一步步调整权重 | 一次性解析求解，直接算出该改多少 |
| 公式 | W = W - lr × ∇L | W_new = W_old + u·v^T |
| 过程 | 需要多轮训练 | 只需要一次矩阵计算 |
| 速度 | 慢（分钟级） | 快（秒级） |

#### 与微调的区别

| | 微调（Fine-tuning） | 知识编辑（ROME） |
|--|--|--|
| 改动范围 | 改动整个模型的大量权重 | 只改一层的一个秩一矩阵 |
| 数据需求 | 需要大量训练数据 | 只需要一条知识（prompt+新答案） |
| 副作用 | 灾难性遗忘（其他知识被破坏） | 极小（理想情况下不影响其他知识） |
| 目标 | 让模型整体学会新能力 | 只改一条特定知识 |
| 比喻 | 重新上一门课 | 在课本上改一个字 |

### Q2: ROME的具体操作步骤是什么？

#### 第一步：因果追踪（Causal Tracing）—— 定位知识在哪

**目的：** 找到"Messi→football"这条知识存储在模型的哪一层。

**原理：** 逐层"破坏"隐藏状态，看哪一层被破坏后模型答错了，说明知识就在那一层。

```
原始输入: "Messi plays the sport of ___"
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  第0层        第17层       第47层
  破坏→还答对   破坏→答错了!  破坏→还答对
                ↑
          知识在第17层！
```

**伪代码：**
```python
for layer in range(num_layers):
    # 1. 正常运行，记录每层的隐藏状态
    hidden_states = model.forward(prompt, output_hidden_states=True)
    
    # 2. 破坏第layer层的subject位置的隐藏状态（加噪声）
    corrupted = hidden_states[layer].clone()
    corrupted[subject_pos] += noise
    
    # 3. 用破坏后的状态继续前向传播
    output = model.forward_from_layer(layer, corrupted)
    
    # 4. 看模型还能不能答对
    if output != "football":
        print(f"知识在第{layer}层！")
        break
```

#### 第二步：计算左向量 u —— "改哪里"

**目的：** 找到权重矩阵中，哪些神经元负责存储"Messi→football"。

**方法：** 用prompt激活模型，看subject位置的隐藏状态。

```python
# 输入: "Lionel Messi plays the sport of"
# 运行模型，拿到subject ("Lionel Messi") 最后一个token的隐藏状态
with torch.no_grad():
    outputs = model(prompt)
    # 提取subject最后一个token的表示
    u = outputs.hidden_states[layer][:, subject_last_pos, :]
    # u 的形状: [hidden_size]，比如 [1600]
```

**含义：** u向量就是"Messi这个概念"在第17层的神经激活模式。

#### 第三步：计算右向量 v —— "改成什么"

**目的：** 计算需要怎么修改权重，才能把输出从"football"变成"basketball"。

**方法：** 解一个优化问题。

```python
# 目标：找到v，使得 W_new @ u 输出 "basketball"
# W_new = W_old + u ⊗ v
# 即：W_old @ u + (u ⊗ v) @ u = target_direction

# target_direction = 模型输出"basketball"时最后一层的表示
target_dir = get_output_direction("basketball")

# 解方程求v
# min ||(W_old + u·v^T) @ u - target||^2
# 这是一个最小二乘问题，有解析解：
v = solve_for_v(W_old, u, target_dir)
```

**数学上的解析解：**
```
v* = (K^T K)^{-1} K^T (target - W_old @ u)
其中 K = u ⊗ I（把u扩展成矩阵）
```

#### 第四步：更新权重 —— 执行手术

**目的：** 把计算好的秩一更新应用到模型上。

```python
# 秩一更新
# W_new[第17层] = W_old[第17层] + u · v^T
# u的形状: [hidden_size]
# v的形状: [hidden_size]  
# u · v^T 的形状: [hidden_size, hidden_size]

layer = 17
model.transformer.h[layer].mlp.c_proj.weight.data += torch.outer(u, v)
```

**关键点：** 只改了一层MLP的一个矩阵，其他层完全不动。

### Q3: ROME涉及哪些代码文件？

```
EasyEdit-main/
├── hparams/ROME/gpt2-xl.yaml          ← 配置文件（模型路径、层号、超参数）
│
├── easyeditor/
│   ├── editors/editor.py               ← 入口：BaseEditor.edit() 方法
│   │
│   └── models/rome/
│       ├── rome_main.py                ← 主流程：apply_rome_to_model()
│       ├── compute_u.py                ← 第二步：计算u向量（定位）
│       ├── compute_v.py                ← 第三步：计算v向量（新内容）
│       ├── repr_tools.py               ← 工具：提取隐藏状态
│       └── trace.py                    ← 第一步：因果追踪，定位知识在第几层
```

| 文件 | 做什么 | 对应步骤 |
|------|--------|---------|
| gpt2-xl.yaml | 配置：模型名、编辑哪一层、超参数 | 准备阶段 |
| editor.py | 接收调用，分发给具体方法 | 入口 |
| rome_main.py | 串联整个流程：trace→u→v→update | 主控 |
| trace.py | 逐层破坏隐藏状态，找到知识在哪层 | 第一步 |
| compute_u.py | 跑模型，提取subject的隐藏状态 | 第二步 |
| compute_v.py | 解方程，算出怎么改权重 | 第三步 |
| repr_tools.py | 辅助工具：拿到指定位置的表示 | 工具 |

#### 调用链路
```
run_edit.py
  └→ editor.edit()                    # editor.py
       └→ apply_rome_to_model()       # rome_main.py
            ├→ compute_u()            # compute_u.py
            │    └→ get_reprs_at_word_tokens()  # repr_tools.py
            ├→ compute_v()            # compute_v.py
            └→ 更新权重矩阵            # rome_main.py
```

### Q4: 四个评估指标具体怎么用？

#### 代码示例

```python
# 评估局部性：编辑梅西后，其他知识有没有被破坏
locality_inputs = {
    'neighborhood': {
        'prompt': ['Joseph Fischhof, the', 'Larry Bird is a professional', 'In Forssa, they understand'],
        'ground_truth': ['piano', 'basketball', 'Finnish']
    }
}

metrics, edited_model, _ = editor.edit(
    prompts=prompts,
    ground_truth=ground_truth,
    target_new=target_new,
    locality_inputs=locality_inputs,
    keep_original_weight=False
)
```

#### 指标详解

| 指标 | 检查什么 | 理想值 | 本例测试方式 |
|------|---------|--------|-------------|
| Reliability | 编辑目标成功了吗？ | ≈1.0 | 问梅西→输出basketball？ |
| Generality | 换种问法也能答对？ | ≈1.0 | "Messi plays which sport"→basketball？ |
| Locality | 不相关知识被破坏了吗？ | ≈1.0 | "Larry Bird plays"→仍basketball？ |
| Portability | 相关推理也更新了吗？ | ≈1.0 | "Messi的运动需要什么装备"→篮球装备？ |

#### 直观对比验证

```python
generation_prompts = ["Lionel Messi, the", "The law in Ikaalinen declares the language"]

# 编辑前
pre_edit_outputs = model.generate(...)

# 编辑后
post_edit_outputs = edited_model.generate(...)
```

**预期结果：**
- "Lionel Messi, the" → 编辑前输出football相关，编辑后输出basketball相关
- "The law in Ikaalinen..." → 编辑前后都输出Finnish（没被影响）

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|

### 实验概述

**目标：** 改变GPT-2-XL的知识记忆，将梅西的职业从足球改成篮球

**步骤：**
1. 安装EasyEdit环境

### 代码实现

```python
import os
os.environ["CUDA_VISIBLE_DEVICES"] = "0"

from easyeditor import BaseEditor, ROMEHyperParams

# ========== 第一步：准备参数 ==========
hparams = ROMEHyperParams.from_hparams('./hparams/ROME/gpt2-xl.yaml')

# ========== 第二步：准备编辑数据 ==========
prompts = ['Question: What sport does Lionel Messi play? Answer:']
ground_truth = ['football']
target_new = ['basketball']
subject = ['Lionel Messi']

# ========== 第三步：实例化Editor ==========
editor = BaseEditor.from_hparams(hparams)

# ========== 第四步：执行编辑 ==========
metrics, edited_model, _ = editor.edit(
    prompts=prompts,
    ground_truth=ground_truth,
    target_new=target_new,
    subject=subject,
    keep_original_weight=False
)

# ========== 第五步：查看结果 ==========
print("编辑结果：")
print(metrics)
```

## 四、遇到的问题及解决

### 问题1：requirements.txt版本号不存在
**原因：** torch==2.9.1等版本号是"未来版"，PyTorch最新只到2.8.0
**解决：** 去掉所有版本限制，改成不带版本号的包名

### 问题2：Python 3.8不支持list[str]语法
**原因：** list[str]类型注解需要Python 3.9+
**解决：** 改成List[str]（从typing导入）

### 问题3：模型路径不存在
**原因：** ./hugging_cache/gpt2-xl是本地路径，云平台上不存在
**解决：** 改成"gpt2-xl"，让HuggingFace自动下载

### 问题4：无法访问HuggingFace
**原因：** AutoDL服务器在国内，访问HuggingFace被墙
**解决：** export HF_ENDPOINT=https://hf-mirror.com

### 问题5：transformers版本兼容性问题
**原因：** EasyEdit代码需要特定版本的transformers，但与其他依赖冲突
**解决：** 使用transformers==4.35.0，修改EasyEdit代码中的import语句

### 问题6：dropout()参数类型错误
**原因：** transformers版本与EasyEdit不兼容，模型输出是dict而非tensor
**解决：** 需要找到兼容的transformers版本组合

## 五、关键概念总结

### ROME算法流程

```
输入: "Messi plays ___"
         │
         ▼
┌─────────────────────────────────────────────┐
│  因果追踪: 逐层破坏，找到知识在第17层         │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  计算u: 跑一遍模型，拿到subject的隐藏状态     │
│  u = hidden_states[17][subject_pos]          │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  计算v: 解方程，让输出从football→basketball   │
│  v = solve(W_old, u, target="basketball")    │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  更新权重: W[17] = W[17] + u·v^T            │
│  只改一层，只改一个秩一矩阵                    │
└─────────────────────────────────────────────┘
         │
         ▼
输出: "basketball" ✓
```
