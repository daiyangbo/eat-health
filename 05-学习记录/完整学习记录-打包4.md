## 一、项目概述

### 实验目标
用**SFT（监督微调）**让Qwen2.5-Math-1.5B学习DeepSeek-R1生成的推理回复，从中学习复杂的数学推理技能（如反思、验算等）。

### 核心思路
> 用DeepSeek-R1的解题过程当"教材"，教Qwen2.5-Math-1.5B这个"学生"学会反思、验算等高级推理能力。

### 数据来源
- **训练集**：[DeepMath-103K](https://huggingface.co/datasets/zwhe99/DeepMath-103K)（103,022条高难度数学题，每条带R1的3条推理路径）
- **测试集**：[MATH-500](https://huggingface.co/datasets/zwhe99/MATH)（500道高中竞赛级数学题）
- **基座模型**：[Qwen/Qwen2.5-Math-1.5B](https://huggingface.co/Qwen/Qwen2.5-Math-1.5B)（1.5B参数，未SFT）

### 环境要求
- Python 3.8+
- PyTorch + Transformers + Datasets + vllm
- 至少40GB显存的GPU（推荐A100 80GB）
- 磁盘预留50GB+

## 五、核心代码片段

### 5.1 环境配置

```python
import transformers
import torch
import datasets
from datasets import load_dataset
from transformers import AutoTokenizer, AutoModelForCausalLM, Trainer, TrainingArguments, DataCollatorForSeq2Seq
import matplotlib.pyplot as plt
from vllm import LLM, SamplingParams
import copy
```

### 5.2 数据加载与筛选

```python
# 加载数据集
data = load_dataset("zwhe99/DeepMath-103K")

# 筛选：只保留答案是纯数字的
data_with_number_answer = data["train"].filter(
    lambda x: x["final_answer"].isdigit()
)

# Tokenize：把question + R1推理拼成对话格式
def tokenize(example):
    question = example["question"]
    r1_solution = example["r1_solution_1"]
    message = [
        {"role": "user", "content": question},
        {"role": "assistant", "content": f"<think>\n{r1_solution}\n"}
    ]
    result = qwen_tokenizer.apply_chat_template(
        message, add_generation_prompt=False, tokenize=True, return_dict=True
    )
    result["length"] = len(result["input_ids"])
    result["labels"] = result["input_ids"].copy()
    return result

train = data_with_number_answer.map(tokenize, batched=False, num_proc=16)

# 长度筛选：只保留 < 4096 的
train = train.filter(lambda x: x["length"] < 4096, num_proc=16)
# 最终 15,310 条
```

### 5.3 模型加载与训练

```python
# 加载基座模型
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-Math-1.5B",
    torch_dtype=torch.bfloat16,
    device_map="auto"
)

# 训练参数
training_args = TrainingArguments(
    per_device_train_batch_size=4,      # 显存不足设为1
    gradient_accumulation_steps=16,     # 有效batch=64
    num_train_epochs=3,
    learning_rate=1e-6,
    lr_scheduler_type="cosine",
    output_dir="./checkpoints",
    save_strategy="steps",
    save_steps=100,
    save_total_limit=1,
    bf16=True,
    group_by_length=True,
    torch_compile=True,
    gradient_checkpointing=False,       # 显存不足设为True
    seed=42,
    data_seed=42
)

# 训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train,
    processing_class=qwen_tokenizer,
    data_collator=DataCollatorForSeq2Seq(qwen_tokenizer, padding=True, pad_to_multiple_of=8, return_tensors="pt"),
)
trainer.train()

# 保存模型
trainer.save_model("./checkpoints/final_model")
```

### 5.4 推理生成

```python
# 加载测试集
math500 = load_dataset("zwhe99/MATH", split="math500")

# 拼成对话格式
data = []
for line in math500:
    line = copy.deepcopy(line)
    line["messages"] = [{"role": "user", "content": line["problem"]}]
    data.append(line)

# 用vllm推理
base_model = "./checkpoints/final_model"
llm = LLM(model=base_model, tensor_parallel_size=1, max_model_len=4096)
tokenizer = AutoTokenizer.from_pretrained(base_model)

# 拼prompt，添加 <think> 触发思考
prompts = [
    tokenizer.apply_chat_template(
        conversation=line["messages"],
        tokenize=False,
        add_generation_prompt=True,
    )
    for line in data
]
prompts = [prompt + "<think>" for prompt in prompts]

# 生成（贪婪解码）
sampling_params = SamplingParams(temperature=0, max_tokens=3700)
outputs = llm.generate(prompts, sampling_params)
```

### 5.5 数学评测

```python
from openmathinst_utils import math_equal, extract_answer

# 提取答案
for line, output in zip(data, outputs):
    line["response"] = output.outputs[0].text
    line["extracted_answer"] = extract_answer(output.outputs[0].text)

# 比对答案
for line in data:
    line["correct"] = math_equal(line["extracted_answer"], line["expected_answer"])

print("Accuracy:", sum(line["correct"] for line in data) / len(data))
# Accuracy: 0.452
```

## 六、实验结果

### 训练结果
- 训练数据：15,310条
- 训练轮数：3 epochs
- 模型保存路径：`./checkpoints/final_model`

### 测试结果
- 测试集：MATH-500
- **Accuracy: 45.2%**

### Case Study 观察
微调后的模型出现明显特征：
1. **指令遵循能力**：能按要求逐步推理
2. **反思倾向**：出现大量"confirm"字样
3. **代码验算**：倾向于用Python代码验证答案

示例输出：
```
问题：将循环小数0.1331...化为分数...
推理过程：
1. 设x = 0.1331...
2. 10000x = 1331.1331...
3. 9999x = 1331
4. x = 1331/9999
5. 用欧几里得算法求GCD...（详细步骤）
6. 用Python代码确认：
   from math import gcd
   print(gcd(1331, 9999))  # 输出1
7. 最终答案：11330
```
