## 一、PDF内容概览

### 1. 隐写术简介
- 什么是隐写术：将秘密信息隐藏在公开载体中，使第三方察觉不到信息传递行为的存在
- 与密码术的区别：密码术隐藏信息**内容**，隐写术隐藏信息传递**行为本身**
- 应用场景：隐蔽通信、版权保护、军事商业领域

### 2. 隐写vs密码术对比

| | 密码术 | 隐写术 |
|--|--------|--------|
| 隐藏什么 | 信息的内容 | 信息传递行为的存在 |
| 结果 | 密文（看起来就是加密过的） | 看起来和普通载体一模一样 |
| 痕迹 | 中间人知道有秘密，只是看不懂 | 中间人根本不知道有秘密 |
| 比喻 | 用密码本写信 | 把情报藏在照片里 |

### 3. 文本隐写的分类
- **修改式**：在已有文本中修改某些特征（如字体间距、颜色）嵌入信息
- **生成式**：利用文本生成模型，在生成过程中干预token选择来嵌入信息
- 近年来随着大模型发展，生成式文本隐写已成为主流

### 4. 大模型隐写原理
- LLM生成时返回next token的概率分布（logits表）
- 正常情况下按概率随机采样一个token
- 隐写时根据秘密信息的编码规则，从候选token中选择特定token
- 解码时用相同模型、上文、参数，可重建完全相同的编码空间，反向提取信息

### 5. 编码方式
- **固定顺序编码**：对前n个token按顺序编码（如000, 001, 010...），简单但效率固定
- **霍夫曼编码**：根据token概率动态分配变长编码，高频token用短码，嵌入效率更高

### 6. 隐写的意义与未来
- 优势：利用大模型生成更自然、更难被发现的隐写内容
- 编码空间：不同编码方式可创造不同的隐写术
- 交叉扩展：可迁移到DNA等序列化数据的隐写
- 与水印的关系：KGW水印也在推理时干预token选择，但目标相反
- 隐写分析：可检测网络上的隐性攻击（如阴阳怪气、网暴）

## 二、核心知识点详解

### Q1: 隐写术和密码术的核心区别是什么？

**隐藏的对象不同：**

| | 密码术 | 隐写术 |
|--|--------|--------|
| 隐藏什么 | 信息的**内容** | 信息传递**行为本身的存在** |
| 结果 | 密文（看起来就是加密过的，暴露了"有秘密"） | 看起来和普通载体一模一样 |
| 痕迹 | 中间人知道有人在传秘密，只是看不懂 | 中间人根本不知道有人在传秘密 |

**间谍的例子：**
- 密码术 = 间谍用密码本写信，敌人截获后知道这是密文，只是破译不了
- 隐写术 = 间谍把情报藏在一张普通照片里，敌人截获后觉得就是一张普通照片，根本不会怀疑

---

### Q2: 大模型隐写的编码过程

假设模型返回前8个token，每个对应一个3位二进制编码。要嵌入二进制串`01011001`：

| 轮次 | 切出的3位 | 选哪个编码的token |
|------|----------|------------------|
| 第1轮 | `010` | 编码为`010`的token（第3个，0起） |
| 第2轮 | `110` | 编码为`110`的token（第7个） |
| 第3轮 | `01` | 只剩2位，补0为`010`，选编码`010`的token |

共3轮。最后一轮不足k位需要补零，接收方需提前知道总比特数才能正确解码。

---

### Q3: 霍夫曼编码 vs 固定长度编码（FLC）

**"效率"指什么：** 每生成一个token平均能嵌入多少比特（bits/token）。

**霍夫曼更高效的原因：** 给高概率token分配短码，低概率token分配长码。

| Token | 概率 | 固定编码(FLC, k=2) | 霍夫曼编码 |
|-------|------|-------------------|-----------|
| "the" | 0.7 | `00`（2位） | `0`（1位） |
| "a" | 0.2 | `01`（2位） | `10`（3位） |
| "an" | 0.08 | `10`（2位） | `110`（3位） |
| "one" | 0.02 | `11`（2位） | `111`（3位） |

- FLC平均：每个token固定嵌入2 bits
- 霍夫曼平均：0.7×1 + 0.2×3 + 0.08×3 + 0.02×3 = 1.6 bits/token

**差距最大的情况：** 当概率分布极度倾斜时（一个token概率0.99，其他分0.01），霍夫曼给高频token只分配1位，效率接近每token嵌入1 bit。FLC仍要给每个token分配k位，浪费了概率分布中隐含的编码潜力。

**反之**，如果所有token概率完全相等（均匀分布），两种编码效率相同。

### Q4: KGW水印 vs 隐写术

**设计目标的根本不同：**

| | KGW水印（Ch5） | 隐写术（Ch7） |
|--|----------------|--------------|
| 目标 | 让接收方能**检测到**"这段文本是AI生成的" | 让接收方能**提取出**一段具体的秘密信息 |
| 对第三方 | 水印信号要**尽量强**，即使被改写也能检测到 | 隐写痕迹要**尽量弱**，第三方完全察觉不到 |
| 信息内容 | 不编码具体信息，只是一个统计"签名" | 编码一段具体的二进制数据 |
| 比喻 | 在钞票上盖章——大家都能看到章 | 把情报藏在照片里——只有收件人知道 |

**编码方式的差异：**
- KGW不关心编码效率，只需要把token分成绿/红两组，给绿组加buff
- 隐写极度关心编码效率，用霍夫曼/FLC最大化每个token的信息载荷

**关键矛盾：**
- 水印希望token选择**有偏**（绿名单token被明显选中更多）→ 容易被检测
- 隐写希望token选择**无偏**（统计分布和正常文本完全一样）→ 不被发现

同样是干预token采样：水印是**故意留下痕迹**，隐写是**尽量不留痕迹**。

### Q5: "阴阳怪气"是一种隐写

**为什么成立：** 隐写的核心结构是**表面内容 ≠ 真实意图**，且第三方检测系统看不出真实意图。

| 隐写要素 | 代码实现 | "阴阳怪气" |
|---------|----------|-----------|
| 载体 | GPT-2生成的正常文本 | 一句看起来正面/中性的话 |
| 隐藏的信息 | 二进制秘密数据 | 讽刺、攻击的真实含义 |
| 检测方 | 隐写分析器 | 关键词过滤系统 |
| 绕过方式 | 让文本统计分布正常 | 不使用任何敏感词 |

**相同点：** 都把真实信息藏在看起来无害的载体中，都存在检测方，都利用检测方的盲区。

**不同点：**

| | 代码实现 | 阴阳怪气 |
|--|---------|---------|
| 编码方式 | 数学算法，确定性 | 人类语感、文化背景，模糊 |
| 载体 | LLM生成的token序列 | 人类语言 |
| 解码 | 用相同模型+参数可完美还原 | 需要人类理解能力 |
| 检测 | 统计方法可检测（z-score） | 需要深层语义理解 |

**本质联系：** 隐写分析技术未来有可能用来自动识别网络上的隐性网暴和阴阳怪气。

class Huffman():
    def __init__(self, k=None, bits=None):
        self.k = k           # 候选词数量
        self.bits = bits      # 要嵌入的比特串

    def __call__(self, pred_prob):
        # 1. 取前k个token的概率
        prob, idx = torch.topk(pred_prob, k=self.k, dim=1)

        root = createHuffmanTree(nodes)

        # 3. 从比特串中取出前几位，找编码匹配的token
        for i, code in enumerate(codes):
            bit = self.bits[:len(code)]
            if bit == code:
                bit_word_index = word_prob[i][0]
                self.bits = self.bits[len(code):]  # 消耗比特

        return bit_word_index

#### FLC编码核心逻辑（Cell2）

```python
class FLC():
    def __init__(self, k=None, bits=None):
        self.k = k            # 每token嵌入的比特数
        self.bits = bits       # 要嵌入的比特串

    def __call__(self, pred_prob):
        # 1. 取前2^k个token
        prob, idx = torch.topk(pred_prob, k=2**self.k, dim=1)
        # 2. 给每个token分配固定k位编码
        codes = [str(bin(i))[2:].zfill(self.k) for i in range(2**self.k)]
        # 3. 取比特串前k位，找编码匹配的token
        bit = self.bits[:self.k]
        for i, code in enumerate(codes):
            if bit == code:
                bit_word_index = word_prob[i][0]
                self.bits = self.bits[self.k:]  # 消耗k位
                break
        return bit_word_index
```

### 生成函数逻辑

```python
def generate_text_with_steganography(model, tokenizer, prompt, handle, max_length=30):
    for _ in range(max_length):
        if len(handle.bits) == 0:
            break
        # 模型预测 → 概率分布
        outputs = model(stega_output)
        next_token_probs = F.softmax(outputs.logits[:, -1, :], dim=-1)

        # 隐写分支：用编码算法选token
        bit_word_index = handle(next_token_probs)
        stega_output = torch.cat([stega_output, bit_word_index.unsqueeze(1)], dim=1)

        # 普通分支：按概率随机采样
        next_token = torch.multinomial(next_token_probs, num_samples=1)
        normal_output = torch.cat([normal_output, next_token], dim=1)
```
