## 一、项目概述

### 实验目标
在LLM生成的文本中嵌入不可见但可算法检测的水印，并评估检测性能和鲁棒性。

### 核心思路
> 生成时偷偷给一半词（绿名单）加优先权，检测时数这些词是否异常多。多得离谱 → 有水印。

### 工具来源
- **X-SIR仓库**：https://github.com/zwhe99/X-SIR
- 三种水印算法：KGW、SIR、X-SIR
- 两种攻击方法：paraphrase、translation

### 数据与模型
- **数据**：`data/dataset/mc4/mc4.en.jsonl`（500条prompt）
- **模型**：`baichuan-inc/Baichuan-7B`
- **水印算法**：KGW

### 环境要求
- Python 3.10
- PyTorch 2.1+ / CUDA 11.8
- GPU：RTX 3090（24GB）或 A100

## 二、水印原理详解

### 2.1 KGW算法：生成阶段（嵌入水印）

LLM每一步输出一个概率分布，然后采样一个token。KGW在采样前插入一步：

```
模型输出 logits（未归一化的分数）
    ↓
Step 1: 用前一个token当种子，哈希出"绿名单"（约50%的词）
Step 2: 给绿名单token的logits加上常数δ
Step 3: softmax → 概率分布（绿名单token概率被抬高）
Step 4: 采样 → 绿名单token更容易被选中
```

**具体示例：**
```
原始logits:  "the"=3.2, "a"=2.8, "cat"=1.5, "dog"=1.2

hash("前一个token") → 绿名单 = {"the", "cat", "dog"}

加buff(δ=1.0):  "the"=4.2🟢, "a"=2.8🔴, "cat"=2.5🟢, "dog"=2.2🟢

softmax后:      "the"=36%, "a"=9%, "cat"=6%, "dog"=5%

采样 → 更可能选中"the"（绿名单token）
```

### 2.2 KGW算法：检测阶段（找水印）

检测器不需要访问模型，只需要tokenizer和同样的哈希函数：

```
文本: "the cat sat on the mat"
       ↓ 分词
token序列: [the, cat, sat, on, the, mat]
       ↓ 逐个检查
对每个token：
  1. 用前一个token哈希出绿名单（和生成时一模一样）
  2. 当前token在绿名单里吗？
  3. 统计绿token数量

绿token数 / 总token数 → 绿token比例
       ↓ 计算z-score
z = (观察到的绿token数 - 期望值) / 标准差
```

**检测判据：**
- 无水印文本：绿token比例 ≈ 50% → z ≈ 0~2
- 有水印文本：绿token比例 > 50% → z ≈ 10+
- z > 阈值（约4）→ 判定有水印

### 2.3 为什么能重建绿名单？

| 条件 | 说明 |
|------|------|
| 哈希函数公开 | 代码里写死了，不需要密钥 |
| 前一个token已知 | 文本在手里，分词后就能看到 |
| 确定性 | 同一个输入 → 同一个输出 |

**检测时不需要知道δ是多少**，只需要判断绿token比例是否异常高。

### 2.4 为什么需要上下文相关的绿名单？

如果绿名单固定不变，攻击者很容易把绿名单的词全换成红名单的词来去水印。所以**每生成一个词，绿名单都变一次**（基于前一个token的哈希），这样：
- 水印嵌入是上下文相关的，更难去除
- 检测时也需要用同样的哈希方式重建绿名单

## 三、鲁棒性原理

### 3.1 为什么需要测试鲁棒性？

水印在"干净"环境下检测效果好，但现实中文本可能被改写。鲁棒性测试回答：
> 如果有人故意改写带水印的文本，水印还能被检测到吗？

### 3.2 攻击方式

| 攻击 | 做法 | 原理 |
|------|------|------|
| 翻译 | 英→中→英，或直接英→中 | 换了一套词，绿名单token全换了 |
| 改写（Paraphrase） | 用另一个LLM重新表述 | 用不同的词表达相同意思，绿名单token被替换 |

### 3.3 为什么改写能削弱水印？

```
原始文本: the(红) cat(绿) sat(绿) on(红) the(绿) mat(绿)
绿token比例: 4/6 = 67%  → z-score高 → 检测到水印

翻译/改写后: 一(?) 只(?) 猫(?) 坐(?) 在(?) 垫子(?)
绿token比例: ≈50%（随机水平）→ z-score低 → 检测不到水印
```

改写后token序列完全变了 → 前一个token变了 → 绿名单变了 → 水印信号被"稀释"。

### 3.4 增强鲁棒性的主流方法

| 方法 | 水印层面 | 鲁棒性 | 原理 |
|------|----------|--------|------|
| KGW | token级 | 低 | 在token采样时加buff，换词就失效 |
| SIR | 语义级 | 中 | 在模型隐藏状态上加偏置，同义词替换不影响 |
| X-SIR | 语义+上下文 | 高 | 用上下文动态选择偏置方向，攻击者无法预测 |
| SemStamp | 语义空间 | 高 | 把句子映射到语义空间嵌入水印，对措辞不敏感 |
| 多层水印 | 混合 | 最高 | 同时在token/句法/语义多个层面嵌入 |

**核心趋势：** 从token级 → 语义级，鲁棒性越来越强，但实现复杂度也越高。

## 四、实验完整流程

### 实验概述

**目标：** 用KGW算法在Baichuan-7B生成的文本中嵌入水印，检测并评估性能。

**步骤：**
1. 环境准备：克隆X-SIR，创建conda环境
2. 数据准备：确认prompt数据文件（500条）
3. 水印嵌入：gen.py生成带水印文本
4. 水印检测：detect.py计算z-score
5. 水印评估：eval_detection.py计算AUC和ROC曲线
6. 鲁棒性评估（可选）：翻译攻击后重新检测

### 实际执行命令

#### Step 1: 环境准备
```bash
source /etc/network_turbo
export HF_ENDPOINT=https://hf-mirror.com

cd /root
git clone https://github.com/zwhe99/X-SIR
cd X-SIR

conda create -n xsir python==3.10.10 -y
conda activate xsir

# 去掉版本号装依赖
cp requirements.txt requirements.txt.bak
sed -i 's/==.*//g' requirements.txt
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 降级兼容PyTorch 2.1
pip install transformers==4.35.0 -i https://pypi.tuna.tsinghua.edu.cn/simple
pip install sentence_transformers==2.2.2 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### Step 2: 数据准备
```bash
wc -l data/dataset/mc4/mc4.en.jsonl
# 输出: 500
```

#### Step 3: 水印嵌入
```bash
MODEL_NAME=baichuan-inc/Baichuan-7B
MODEL_ABBR=baichuan-7b
WATERMARK_METHOD_FLAG="--watermark_method kgw"

python3 gen.py \
    --base_model $MODEL_NAME \
    --fp16 \
    --batch_size 32 \
    --input_file data/dataset/mc4/mc4.en.jsonl \
    --output_file gen/$MODEL_ABBR/kgw/mc4.en.mod.jsonl \
    $WATERMARK_METHOD_FLAG
```

#### Step 4: 水印检测
```bash
# 检测有水印文本
python3 detect.py \
    --base_model $MODEL_NAME \
    --detect_file gen/$MODEL_ABBR/kgw/mc4.en.mod.jsonl \
    --output_file gen/$MODEL_ABBR/kgw/mc4.en.mod.z_score.jsonl \
    $WATERMARK_METHOD_FLAG

# 检测无水印文本
python3 detect.py \
    --base_model $MODEL_NAME \
    --detect_file data/dataset/mc4/mc4.en.jsonl \
    --output_file gen/$MODEL_ABBR/kgw/mc4.en.hum.z_score.jsonl \
    $WATERMARK_METHOD_FLAG
```

#### Step 5: 水印评估
```bash
python3 eval_detection.py \
    --hm_zscore gen/$MODEL_ABBR/kgw/mc4.en.hum.z_score.jsonl \
    --wm_zscore gen/$MODEL_ABBR/kgw/mc4.en.mod.z_score.jsonl \
    --roc_curve roc
```

#### Step 6: 鲁棒性评估（可选）
```bash
# 翻译攻击
python3 attack/translate.py \
    --input_file gen/$MODEL_ABBR/kgw/mc4.en.mod.jsonl \
    --output_file gen/$MODEL_ABBR/kgw/mc4.en-zh.mod.jsonl \
    --model gpt-3.5-turbo-1106 \
    --src_lang en \
    --tgt_lang zh

# 重新检测和评估
python3 detect.py \
    --base_model $MODEL_NAME \
    --detect_file gen/$MODEL_ABBR/kgw/mc4.en-zh.mod.jsonl \
    --output_file gen/$MODEL_ABBR/kgw/mc4.en-zh.mod.z_score.jsonl \
    $WATERMARK_METHOD_FLAG

python3 eval_detection.py \
    --hm_zscore gen/$MODEL_ABBR/kgw/mc4.en.hum.z_score.jsonl \
    --wm_zscore gen/$MODEL_ABBR/kgw/mc4.en-zh.mod.z_score.jsonl \
    --roc_curve roc_attacked
```

## 六、遇到的问题及解决

### 问题1：scipy==1.12.0安装失败
**原因：** Python 3.8不支持scipy 1.12.0
**解决：** 选择Python 3.10的镜像

### 问题2：PyTorch 2.0只能选Python 3.8
**原因：** AutoDL镜像限制
**解决：** 选择PyTorch 2.1.2 + Python 3.10 + CUDA 11.8

### 问题3：transformers版本不兼容PyTorch 2.1
**原因：** 新版transformers要求PyTorch >= 2.4
**解决：** 降级 `pip install transformers==4.35.0`

### 问题4：sentence_transformers导入UdopConfig失败
**原因：** 新版sentence_transformers依赖新版transformers
**解决：** 降级 `pip install sentence_transformers==2.2.2`

### 问题5：`No module named 'src_watermark.uw.test'`
**原因：** X-SIR仓库代码中`__init__.py`导入了不存在的test模块
**解决：** 注释掉`src_watermark/uw/__init__.py`中的`from .test import *`

### 问题6：HuggingFace下载超时
**原因：** 国内访问HuggingFace不稳定
**解决：** `export HF_ENDPOINT=https://hf-mirror.com`

## 八、术语表

| 术语 | 全称 | 含义 |
|------|------|------|
| KGW | - | 一种token级文本水印算法 |
| SIR | - | 一种语义级文本水印算法 |
| X-SIR | - | 上下文感知的语义水印算法 |
| 绿名单 | Green List | 被加buff的token集合，约占词表50% |
| 红名单 | Red List | 未被加buff的token集合 |
| δ | Delta | 给绿名单token加的buff值 |
| z-score | - | 水印强度指标，越高越可能有水印 |
| AUC | Area Under Curve | ROC曲线下面积，检测能力指标 |
| TPR | True Positive Rate | 真正率（检出率） |
| FPR | False Positive Rate | 假正率（误报率） |
| ROC | Receiver Operating Characteristic | 受试者工作特征曲线 |
| Paraphrase | - | 改写攻击，用同义词/不同句式重述 |
| Translation | - | 翻译攻击，通过语言转换削弱水印 |
| Logits | - | 模型输出的未归一化分数 |
| Softmax | - | 将logits转成概率分布的函数 |
