## 🧾 统一数学模型（最终形态）

### 决策变量

```python
# 基础定义
x_i ∈ {0, 0.5, 1, 2}

# 工程实现：根据 allow_half 字段动态调整
if dish_i.allow_half:
    x_i ∈ {0, 0.5, 1, 2}    # 可以打半份（如米饭）
else:
    x_i ∈ {0, 1, 2}          # 不可打半份（如红烧鸡腿）
```

### 输入数据

每个菜：
```json
{
  "dish_id": 1,
  "name": "红烧鸡腿",
  "protein_i": 25,
  "fat_i": 18,
  "carb_i": 5,
  "calories_i": 260,
  "category_i": "蛋白",
  "satiety_i": 0.8,
  "oil_i": "high",
  "availability_i": 1,
  "price_i": 8.0,
  "allow_half": false
}
```

### 目标函数

```
min  F(x, state)
```

展开为：

```
F(x, s) = |P(x) - P*(s)|₁ + |F(x) - F*(s)|₁ + |C(x) - C*(s)|₁  # 营养误差
        + R(x, s)                                                     # 用户偏好损失
        + T(x)                                                        # 时间稳定性惩罚
        + budget_penalty(x, s)                                        # 预算惩罚（MVP软约束）
```

### 约束系统

```
s.t.
  # Hard Constraints（硬约束）
  ∑ x_i (主食) ≥ 1
  ∑ x_i (蛋白) ≥ 1
  ∑ x_i (蔬菜) ≥ 1
  3 ≤ ∑ x_i ≤ 6
  x_i ≤ availability_i
  |Calories(x) - Calories*(s)| ≤ ε
  |x_t - x_{t-1}| ≤ δ

  # Soft Constraints（软约束，MVP阶段）
  budget_penalty(x, s) → 目标函数惩罚项

  # 变量约束
  x_i ∈ {0, 0.5, 1, 2}  # 根据 allow_half 动态调整
```

## ⚠️ 三个工程落地补丁

### 补丁1：预算约束"软化"

#### 问题

预算约束写成 Hard Constraint 会导致无可行解：

```
∑ (x_i · price_i) ≤ Budget(s)
```

如果今天蛋白类菜品都涨价或只剩高价菜，求解器会报错 "Infeasible"

#### MVP 补丁

**MVP V0.5（纯算法版）**：将预算约束改为目标函数中的惩罚项

```python
def budget_penalty(x, prices, budget):
    """
    预算惩罚函数

    允许超预算，但超得越多，F(x,s) 得分越差
    """
    total_cost = sum(x_i * price_i for x_i, price_i in zip(x, prices))
    over_budget = max(0, total_cost - budget)

    # 超预算惩罚（超得越多，惩罚越大）
    return over_budget * 10  # 权重可调
```

**V1.0+（接入 LLM 后）**：无解时 LLM 输出提示

```
"今日套餐超出预算 3 元，建议增加 5 元预算或更换窗口。"
```

#### 目标函数更新

```
F(x, s) = 营养误差 + R(x,s) + T(x) + budget_penalty(x, s)
```

### 补丁2：决策变量"半份"问题

#### 问题

国内大学食堂绝大多数菜品不卖"半份"（尤其是红烧鱼、鸡腿这类按个卖的菜）

#### MVP 补丁

在菜品数据表 `dishes.csv` 中新增一列 `allow_half`（布尔值）

```csv
dish_id,name,protein,fat,carbs,calories,category,satiety,oil,availability,price,allow_half
1,红烧鸡腿,25,18,5,260,蛋白,0.8,high,1,8.0,false
2,米饭,3,0.5,40,175,主食,0.6,low,1,1.0,true
3,清炒西兰花,3,2,5,45,蔬菜,0.5,low,1,3.0,true
```

#### 代码实现

```python
def build_variables(solver, dishes):
    """
    构建决策变量

    根据 allow_half 字段动态调整定义域
    """
    x = []
    for dish in dishes:
        if dish['allow_half']:
            # 可以打半份（如米饭、蔬菜）
            var = solver.IntVar(0, 2, f"x_{dish['id']}")
        else:
            # 不可打半份（如红烧鸡腿）
            var = solver.IntVar(0, 2, f"x_{dish['id']}")

        # 注意：这里用 IntVar，半份通过 0.5 份量系数处理
        x.append(var)

    return x
```

#### 效果

- ✅ 米饭可以打半份
- ✅ 红烧鸡腿不会配出"半条"
- ✅ 求解器不会产生诡异结果

### 补丁3：OR-Tools 求解绝对值目标函数

#### 问题

OR-Tools 不支持直接写 `abs()`，需要引入辅助变量线性化

#### MVP 代码补丁

```python
from ortools.linear_solver import pywraplp

def build_nutrition_loss(solver, dishes, x, target):
    """
    构建营养损失函数

    使用辅助变量将绝对值线性化
    """
    # 计算实际营养素
    protein_sum = sum(x_i * dish['protein'] for x_i, dish in zip(x, dishes))
    fat_sum = sum(x_i * dish['fat'] for x_i, dish in zip(x, dishes))
    carbs_sum = sum(x_i * dish['carbs'] for x_i, dish in zip(x, dishes))

    # 对每个营养素引入两个非负松弛变量
    p_pos = solver.NumVar(0, solver.infinity(), 'p_pos')
    p_neg = solver.NumVar(0, solver.infinity(), 'p_neg')
    f_pos = solver.NumVar(0, solver.infinity(), 'f_pos')
    f_neg = solver.NumVar(0, solver.infinity(), 'f_neg')
    c_pos = solver.NumVar(0, solver.infinity(), 'c_pos')
    c_neg = solver.NumVar(0, solver.infinity(), 'c_neg')

    # 约束：实际值 - 目标值 = 正偏差 - 负偏差
    solver.Add(protein_sum - target['protein'] == p_pos - p_neg)
    solver.Add(fat_sum - target['fat'] == f_pos - f_neg)
    solver.Add(carbs_sum - target['carbs'] == c_pos - c_neg)

    # 营养损失 = 正偏差 + 负偏差（等价于绝对值）
    nutrition_loss = p_pos + p_neg + f_pos + f_neg + c_pos + c_neg

    return nutrition_loss
```

#### 完整目标函数构建

```python
def build_objective(solver, dishes, x, state):
    """
    构建完整目标函数
    """
    # 营养损失
    nutrition_loss = build_nutrition_loss(solver, dishes, x, state['target_PFC'])

    # 预算惩罚
    budget_pen = budget_penalty(x, [d['price'] for d in dishes], state['budget'])

    # 口味疲劳惩罚
    fatigue_pen = fatigue_penalty(x, dishes, state['history_choices'])

    # 最小化目标函数
    objective = solver.Objective()
    objective.SetMinimization()

    # 添加各项（权重可调）
    for i, dish in enumerate(dishes):
        objective.SetCoefficient(x[i], nutrition_loss + budget_pen + fatigue_pen)

    return objective
```

## 🛠️ 技术实现

### MVP V0.5 技术栈

```
Python + OR-Tools

输入：state（用户状态）
  ↓
构建：目标函数 F(x, s) + 约束（含预算软约束）
  ↓
求解：ILP Solver（OR-Tools）
  ↓
输出：菜品组合 x*
```

### 关键代码结构

```python
from ortools.linear_solver import pywraplp

def solve_meal_recommendation(dishes, state):
    """
    求解菜品推荐问题

    参数：
        dishes: 菜品列表（含营养、价格、allow_half等）
        state: 用户状态（目标、预算、历史等）

    返回：
        推荐菜品组合
    """
    solver = pywraplp.Solver.CreateSolver('SCIP')

    # 构建决策变量
    x = build_variables(solver, dishes)

    # 构建约束
    build_constraints(solver, dishes, x, state)

    # 构建目标函数
    objective = build_objective(solver, dishes, x, state)

    # 求解
    status = solver.Solve()

    if status == pywraplp.Solver.OPTIMAL:
        return extract_solution(x, dishes)
    else:
        return None  # 无可行解
```
